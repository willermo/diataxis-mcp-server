# Sample output

This directory contains checked-in example outputs generated by dogfooding the current MVP against this repository's own documentation.

The examples are intended to show the shape and style of the MCP tool outputs. They are not authoritative documentation refactoring instructions.

Current examples:

- `audit-tree.json`: sample output from auditing selected repository documentation as a documentation tree with `scope: "sample"`.
- `refactor-plan.json`: sample advisory refactor plan generated from the same input set.

The corresponding explanation page is:

- `docs/explanation/self-audit.md`

## How to interpret these files

The outputs are produced by deterministic MVP heuristics. They should be read as signals for maintainer review, not as automatic claims that the documentation is wrong.

The tree audit example uses `scope: "sample"`, so coverage warnings describe only the sampled input files. Do not treat those warnings as full documentation coverage findings.

Repository-level files such as `README.md`, `CONTRIBUTING.md`, `SECURITY.md`, `LICENSING.md`, and `NOTICE.md` often serve multiple practical purposes. Mixed-mode findings in these examples may indicate either genuine documentation design opportunities or limitations in the current heuristics.

## Regenerating examples

When core audit or refactor behavior changes, regenerate these examples and update `docs/explanation/self-audit.md` if the interpretation changes.

Before committing regenerated output, run:

```sh
npm run build
npm test
```
